# 生产环境资源配置建议

更新时间：2026-07-03

## 评估规模

- 客户资料维护量：5000 个客户
- 技师数量：30 人
- 客服数量：4 人
- 使用场景：客户管理、订单维护、服务照片/合同附件上传、数据报表、系统设置
- 预估并发：日常 5-15 人同时在线，峰值 20 人以内

## 推荐正式环境配置

### 方案 A：成本优先，适合上线初期

适合：数据量已知、访问量不大、先正式上线使用。

| 资源 | 推荐配置 |
| --- | --- |
| ECS | 2 vCPU / 4 GB 内存 |
| 系统盘 | ESSD 云盘 80 GB |
| 数据库 | ECS 内 Docker MySQL，单独挂载 100 GB ESSD 数据盘 |
| Redis | ECS 内 Docker Redis，512 MB-1 GB |
| OSS | 标准存储 300 GB 起步 |
| 备份 | 每日数据库备份，保留 7-14 天 |

注意：这个方案能用，但数据库和应用在同一台服务器上，后续迁移和容灾能力弱。正式业务跑起来后建议升级到方案 B。

### 方案 B：正式推荐，稳定可持续迭代

适合：正式环境长期使用，并且测试环境持续迭代。

| 资源 | 推荐配置 |
| --- | --- |
| ECS 应用服务器 | 2 vCPU / 8 GB 内存 |
| ECS 系统盘 | ESSD 云盘 100 GB |
| RDS MySQL | 2 vCPU / 4 GB 内存，100 GB 存储，高可用优先 |
| Redis | 1 GB，可先用 ECS 内 Docker Redis；后续再换云 Redis |
| OSS | 标准存储 500 GB 起步 |
| 备份 | RDS 自动备份，OSS 生命周期归档，ECS 快照 |

这是当前规模下更建议购买的正式配置。5000 个客户的数据本身不大，但服务照片、合同附件、报表统计和后续功能迭代会持续增加负载，2C8G 应用服务器会更稳。

### 方案 C：增长预留，适合半年内明显扩张

适合：客户量可能快速超过 1-2 万，图片上传很频繁。

| 资源 | 推荐配置 |
| --- | --- |
| ECS 应用服务器 | 4 vCPU / 8 GB 或 4 vCPU / 16 GB |
| RDS MySQL | 2 vCPU / 8 GB，200 GB 存储 |
| Redis | 2 GB |
| OSS | 1 TB 起步 |
| 负载均衡 | 有多台 ECS 或正式域名 HTTPS 后再接入 |

## OSS 容量估算

客户基础资料、订单和报表数据非常小，主要容量来自：

- 合同附件：PDF 或图片
- 服务照片：每次最多 10 张
- 后续可能增加的客户档案图片、导入文件、备份文件

估算公式：

```text
OSS 容量 = 客户数 × (合同附件平均大小 + 服务次数 × 每次图片数量 × 单张图片大小)
```

建议按以下三档规划：

| 使用强度 | 假设 | 5000 客户估算 | 建议购买 |
| --- | --- | ---: | ---: |
| 轻量 | 每客户 1 个合同 3 MB，2 次服务，每次 3 张图，每张 1.5 MB | 约 60 GB | 100-200 GB |
| 常规 | 每客户 1 个合同 5 MB，4 次服务，每次 5 张图，每张 2 MB | 约 225 GB | 500 GB |
| 偏重 | 每客户 1 个合同 5 MB，8 次服务，每次 10 张图，每张 3 MB | 约 1.2 TB | 1 TB 起，按月扩容 |

正式环境建议先购买 500 GB OSS 标准存储包。图片如果上传频繁，建议同时开启生命周期规则：

- 0-180 天：标准存储，方便查看
- 180 天后：低频访问存储
- 1 年后：归档或冷归档，按业务合规要求决定

## 测试环境建议

测试环境不需要和正式环境同规格，但必须和正式环境隔离数据库、OSS Bucket 和环境变量。

| 资源 | 推荐配置 |
| --- | --- |
| ECS | 1 vCPU / 2 GB，最低；更稳建议 2 vCPU / 2-4 GB |
| 数据库 | Docker MySQL 或低配 RDS |
| OSS | 50-100 GB |
| 域名 | test 子域名或独立测试域名 |

测试环境可以小一些，但不要共用正式数据库和正式 OSS Bucket，避免测试上传、删除、覆盖影响主数据。

## 当前建议采购清单

如果现在要买正式环境，建议：

1. ECS：2 vCPU / 8 GB，ESSD 100 GB。
2. RDS MySQL：2 vCPU / 4 GB，100 GB，高可用版优先。
3. OSS：500 GB 标准存储包。
4. Redis：先用 ECS Docker Redis 1 GB；系统使用稳定后再换云 Redis。
5. 备份：RDS 每日自动备份，ECS 每周快照，OSS 开生命周期。

如果预算要压低：

1. ECS：2 vCPU / 4 GB。
2. MySQL 暂时放 ECS Docker，单独 100 GB 数据盘。
3. OSS：300 GB。

但正式业务长期使用，不建议数据库长期和应用挤在同一台 2C4G 服务器上。
